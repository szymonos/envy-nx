# Charter - `config-templates` shard (shell / pwsh / oh-my-posh templates)

Templates rendered into user shell profiles via managed blocks. The key concerns are cross-shell parity (the same alias name does the same thing in bash, zsh, and pwsh - users move between shells in the same week and expect consistency) and idempotent rendering (the managed block regenerates byte-identically on every setup, which CI verifies).

## Scope

| File                                          | Role                                                                                |
| --------------------------------------------- | ----------------------------------------------------------------------------------- |
| `.assets/config/shell_cfg/aliases_git.sh`     | git helper aliases and functions                                                    |
| `.assets/config/shell_cfg/aliases_kubectl.sh` | kubectl command aliases                                                             |
| `.assets/config/shell_cfg/aliases_nix.sh`     | nix-specific navigation aliases                                                     |
| `.assets/config/shell_cfg/functions.sh`       | Shell utility functions                                                             |
| `.assets/config/shell_cfg/completions.bash`   | Bash tab completions for `nx` (auto-generated from `nx_surface.json`)               |
| `.assets/config/shell_cfg/completions.zsh`    | Zsh tab completions for `nx` (auto-generated)                                       |
| `.assets/config/pwsh_cfg/profile.ps1`         | PowerShell base profile                                                             |
| `.assets/config/pwsh_cfg/profile_nix.ps1`     | PowerShell Nix-managed profile (dot-sourced from `$PROFILE`)                        |
| `.assets/config/pwsh_cfg/_aliases_common.ps1` | Common PowerShell aliases and functions                                             |
| `.assets/config/pwsh_cfg/_aliases_linux.ps1`  | Linux-specific PowerShell aliases                                                   |
| `.assets/config/pwsh_cfg/_aliases_nix.ps1`    | Nix-specific PowerShell aliases (also hosts the fnm self-heal block)                |
| `.assets/config/omp_cfg/*.omp.json`           | Oh My Posh prompt theme configurations (4 themes: base, nerd, nerdplain, powerline) |

**Out of scope:** the rendering machinery itself (lives in `nx_profile.sh` → `nx-cli` shard); the `nx` completer logic (generated artifact, source of truth is `nx_surface.json` → `precommit-hooks` shard for the generator).

## What "good" looks like

- **Cross-shell alias parity.** An alias defined in both bash/zsh and pwsh forms (e.g., `ll` listing files, `gst` for `git status`) does the conceptually same thing on both sides. Divergence is a footgun.
- **No platform-hard-coded paths.** Templates that reference `/usr/local/bin` or `~/Library/...` directly will break when rendered on the wrong platform. Resolve via env vars or runtime detection.
- **PowerShell files use OTBS brace style, 4-space indent, approved verbs** (per [`CLAUDE.md` → PowerShell Style](../../../.claude/CLAUDE.md)). Function names: `Verb-Noun` PascalCase. Local vars: camelCase. Parameters: PascalCase.
- **Shell files are sourceable in both bash and zsh** without errors. The `check_zsh_compat` hook covers most of this; review checks for subtler cases (e.g., functions that test `[[ -v VAR ]]` which behaves differently in zsh strict mode).
- **`omp_cfg/*.omp.json` are valid against oh-my-posh's schema** and don't reference Nerd Font glyphs in non-Nerd themes (`base.omp.json` should be plain-ASCII compatible).
- **`_aliases_*.ps1` files are independently dot-sourceable** in any order. None depend on side effects of another in the set.
- **Auto-generated files have a comment header** stating their generator (`# Generated by tests/hooks/gen_nx_completions.py - do not edit by hand`).
- **The fnm self-heal block in `_aliases_nix.ps1`** stays consistent with the bash/zsh equivalent in `nx_profile.sh`'s `_nx_render_nix_block`.

## What NOT to flag

- **The choice of oh-my-posh + starship over oh-my-zsh.** See [`docs/decisions.md` → "Why oh-my-posh and starship, not oh-my-zsh"](../../../docs/decisions.md#why-oh-my-posh-and-starship-not-oh-my-zsh). Findings suggesting "use a different prompt framework" are out of scope.
- **The four omp themes (base, nerd, nerdplain, powerline).** User choice. Don't suggest pruning to one or adding a fifth.
- **Specific alias names.** These are user-visible and changing them breaks muscle memory. Flag a *broken* alias, not a stylistic preference about the name.
- **The `_aliases_*` underscore prefix on pwsh files.** Convention indicates "dot-sourced from another file"; don't suggest renaming.
- **Long alias lists.** They're long because users want them. Adding aliases is fine; flag genuine *wrong* behavior in one.
- **Anything already in [`design/reviews/accepted.md`](../accepted.md).**

## Severity rubric

| Level    | Definition                                                                                                                                       | Examples                                                                                                                                                                           |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| critical | An alias or function destructively mutates the user's data when invoked with no/wrong args; a profile file fails to source and bricks the shell. | `alias rm='rm -rf'`; `profile_nix.ps1` syntax error that prevents PowerShell from starting.                                                                                        |
| high     | Cross-shell alias diverges in observable behavior; auto-generated file is hand-edited and drifts from generator; missing platform guard.         | `gst` does `git status -s` in bash but `git status` in pwsh; bash completer block doesn't match `nx_surface.json`.                                                                 |
| medium   | OTBS / approved-verb violation; alias works but emits a warning on one shell; theme uses Nerd glyphs in non-Nerd file.                           | PowerShell function uses `Get-FooBar` where `Foo` isn't an approved verb; `base.omp.json` references a glyph only Nerd Fonts have.                                                 |
| low      | Comment typo, alias defined but never likely to be used by anyone, naming inconsistency between sibling alias files.                             | `# Aliases for git` comment left in place after the file was renamed; `gst-all` defined alongside `gst` (the documented one); `aliases_git.sh` uses `kebab-case` for one variable. |

## Categories

| Category        | Use for                                                                             |
| --------------- | ----------------------------------------------------------------------------------- |
| correctness     | An alias or function does the wrong thing for a reasonable input.                   |
| security        | Alias that runs arbitrary remote code, weakens permissions, exposes secrets.        |
| maintainability | Cross-shell drift; convention violation; hidden coupling between alias files.       |
| testability     | A function in `functions.sh` lacks a bats test for a non-obvious code path.         |
| docs            | Comment in template wrong; user-facing alias lacks a one-line `# what` description. |
